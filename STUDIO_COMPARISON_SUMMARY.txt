3. 参数级对比 - 兼容性
================================================================================

创建参数对比：
  老SDK:  visibility(str), license(display name), chinese_name, description, 
          sdk_type, sdk_version, base_image, hardware, cover_image

日志参数对比：

  新SDK:  page_num=1, page_size=100(改成50?), keyword, timestamps
  
  ❌ 差异：page_size默认值从100变成50 → 建议改回100

设置参数对比：
  老SDK:  display_name, description, license, cover_image, sdk_type, 
          sdk_version, base_image, hardware, private(bool)
  新SDK:  同上，通过UpdateStudioSettingsPayload TypedDict

  ✅ 兼容：参数名完全相同，新SDK提供类型提示

4. CLI对比 - 命令结构
================================================================================

创建Studio：
  老: modelscope create owner/name --repo_type studio --sdk-type gradio
  新: ms repo create owner/name --repo-type studio [--description "..."]
  
  ⚠️  问题：新CLI缺少--sdk-type等参数支持
  🔴 影响：高，用户无法指定SDK类型

部署/停止：
  老: modelscope studio deploy/stop owner/name
  新: ms deploy/stop owner/name --repo-type studio
  
  ✅ 功能相同，命令路径重组（studio子命令 → 独立命令）

日志：
  老: modelscope studio logs owner/name --type runtime --page-num 1 --page-size 100
  新: ms logs owner/name --repo-type studio --log-type runtime --page 1 --page-size 50
  
  ⚠️  参数名变化：--type → --log-type，--page-num → --page
  ⚠️  默认值变化：--page-size从100改成50

设置：
  老: modelscope studio settings owner/name --display-name "新名" --private
  新: ms settings owner/name --repo-type studio display_name="新名" private=true
  
  🔴 设计完全不同：参数式 → 键值对式，需要完全重写脚本

Secrets：
  老: modelscope studio secret {list|add|update|delete} owner/name [key] [value]
  新: ms secret {list|add|update|delete} owner/name [key] [value]
  
  ✅ 功能兼容，新SDK有--yes选项更安全

6. 新SDK优势汇总
================================================================================

✅ 功能完整性：
   - 新增get_repo()支持查询Studio元数据
   - 支持payload参数自定义部署配置
   - 类型安全：TypedDict类型提示
   - 显式限制：NotImplementedError清楚表达不支持的操作

✅ 架构设计：
   - OpenAPI-First原则：所有操作优先OpenAPI，透明可追踪
   - 统一repo模式：reduce重复代码，易于扩展新repo类型
   - 日志提取robust：支持logs/items/list/data多个键

✅ 用户体验：
   - Delete操作有确认提示（--yes选项）
   - 异常更清晰（ValueError vs InvalidParameter）
